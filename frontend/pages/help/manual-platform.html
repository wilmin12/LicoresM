Visión General de la Plataforma

El módulo M00 — Plataforma & Administración es la base de todo el sistema de Licores Maduro. Aquí se configura la identidad de la empresa, los usuarios que tienen acceso, sus permisos, los parámetros de correo electrónico y la apariencia de la herramienta.

Este módulo está disponible en la sección Administration del menú lateral y es de acceso exclusivo para los roles Admin y SuperAdmin.

Company Settings

Nombre, logo, contacto y datos legales de la empresa que aparecen en toda la plataforma.

Email Configuration

Servidor SMTP para el envío automático de notificaciones y aprobaciones.

Module Approvers

Correos de los aprobadores de cada módulo (Aankoopbon, Cost Calc, Activity Request).

Usuarios y Roles

Crear usuarios, asignar roles y configurar permisos por módulo y sub-módulo.

Email Configuration

Configura el servidor de correo saliente (SMTP) para que el sistema pueda enviar notificaciones automáticas.

Solo SuperAdmin puede ver y modificar esta configuración.
  1. Ir a Administration → Email Config.
  2. Completar los datos SMTP.
  3. Activar el interruptor "Email Enabled" para habilitar el envío.
  4. Usar el botón Test Email para verificar la conexión antes de guardar.
  5. Hacer clic en Save.

Module Approvers

Define los correos electrónicos de las personas que reciben las notificaciones de aprobación para cada módulo del sistema.

  1. Ir a Administration → Module Approvers.
  2. Para cada módulo, ingresar los correos separados por coma (,).
  3. Hacer clic en Save en la card de cada módulo.
MóduloCuándo se notifica
AankoopbonnenCuando una orden de compra es enviada a aprobación (estado PENDING).
Cost CalculationCuando una calculación es confirmada (estado CF) y requiere aprobación.
Activity RequestCuando una solicitud de actividad es enviada para revisión.
Puedes ingresar múltiples correos: gerencia@empresa.com, finanzas@empresa.com
El sistema validará el formato de cada dirección antes de guardar.
Gestión de Usuarios

Administra las cuentas de acceso al sistema. La tabla muestra por defecto solo los usuarios activos.

Acciones disponibles por usuario:

IconoAcciónDescripción
EditarModifica datos del usuario: nombre, correo, rol, estado activo/inactivo.
Cambiar contraseñaEstablece una nueva contraseña para el usuario.
Permisos individualesConfigura permisos específicos para este usuario que sobrescriben los del rol. Ver sección más abajo.
DesactivarEl usuario no puede iniciar sesión pero su historial se conserva.
ActivarRehabilita el acceso de un usuario desactivado.
EliminarElimina permanentemente el usuario. Solo aparece para usuarios inactivos y solo lo puede ejecutar SuperAdmin.
1Crear un usuario nuevo
  1. Ir a Administration → Users.
  2. Clic en + New User.
  3. Completar: username, nombre completo, correo, rol y contraseña inicial.
  4. Guardar. El usuario puede ingresar de inmediato.
2Ver y filtrar usuarios inactivos
  1. Por defecto la tabla muestra solo usuarios activos.
  2. Clic en Show Inactive para ver también los inactivos.
  3. Clic nuevamente en Hide Inactive para ocultarlos.
3Eliminar un usuario permanentemente
  1. Primero desactiva el usuario con el botón .
  2. Clic en Show Inactive para ver el usuario inactivo.
  3. Clic en el botón que aparece en la fila.
  4. Confirmar en el modal. Esta acción no se puede deshacer.
Solo SuperAdmin puede eliminar usuarios. Si el usuario tiene registros de negocio asociados (órdenes, cálculos), el sistema mostrará un error y no permitirá la eliminación.
4Permisos individuales por usuario

Además de los permisos del rol, es posible configurar permisos específicos para un usuario individual que tienen prioridad sobre el rol.

  1. Clic en en la fila del usuario.
  2. Se abre una tabla con todos los módulos y sub-módulos.
  3. Marca la columna "Override" en las filas que deseas personalizar. Las filas con override activo se destacan en amarillo.
  4. Ajusta los permisos (Access, Read, Write, Edit, Delete, Approve) según necesidad.
  5. Clic en Save Overrides.
  6. Para revertir al usuario a los permisos del rol: clic en Clear all overrides.
Los permisos individuales aplican en el próximo login del usuario. Las filas sin "Override" marcado siguen usando los permisos del rol asignado.

Gestión de Roles y Permisos

Los roles definen qué módulos y acciones puede realizar cada usuario.

Roles predefinidos del sistema:

RolDescripción
SuperAdminAcceso total a todo el sistema incluyendo configuración de email y sesiones. No puede ser eliminado.
AdminAcceso a administración (usuarios, roles, company settings, module approvers). Puede aprobar en todos los módulos.
UsuarioAcceso operativo a los módulos según permisos asignados.
PConfigurar permisos de un rol
  1. Ir a Administration → Roles.
  2. Clic en (Permissions) del rol.
  3. Para cada sub-módulo marcar los permisos necesarios (ver tabla abajo).
  4. Guardar. Los cambios aplican en el próximo login de los usuarios con ese rol.
PermisoDescripción
AccessEl usuario puede ver el módulo en el menú y acceder a la página.
ReadPuede consultar y listar registros.
WritePuede crear nuevos registros.
EditPuede modificar registros existentes.
DeletePuede eliminar registros.
Approve especialPuede aprobar o rechazar solicitudes. Aplica solo en Aankoopbonnen y Cost Calculation. Permite que usuarios sin rol Admin puedan aprobar en esos módulos.
Columna "All": marca o desmarca todos los permisos base (Access → Delete) de una fila de forma rápida. El permiso Approve se configura de forma independiente.
Los permisos de SuperAdmin no pueden modificarse. Siempre tienen acceso total.
Sesiones Activas

Permite ver en tiempo real qué usuarios están conectados al sistema.

Solo SuperAdmin puede ver esta sección.
  1. Ir a Administration → Active Users.
  2. Se muestra la lista de sesiones activas: usuario, rol, IP, fecha/hora de login.
  3. Para cerrar una sesión remota: clic en . El usuario será desconectado en su próxima acción.
Las sesiones inactivas se limpian automáticamente. Este panel muestra las sesiones con token JWT aún válido.
Temas de Color

Cada usuario puede personalizar el color principal de la interfaz. La preferencia se guarda localmente en el navegador.

Los temas disponibles son:

  • Wine — Rojo vino (predeterminado de Licores Maduro)
  • Navy — Azul marino
  • Forest — Verde bosque
  • Ocean — Azul océano
¿Cómo cambiar el tema?
En la parte inferior del sidebar hay cuatro círculos de color. Haz clic en el color deseado. El cambio es inmediato y se recuerda al reabrir el navegador.
Idioma (EN / ES)

El sistema soporta dos idiomas: English (EN) y Español (ES).

Para cambiar el idioma:

  1. En la barra superior (topbar) busca los botones EN ES.
  2. Clic en el idioma deseado. La interfaz se traduce inmediatamente.
  3. La preferencia se guarda en el navegador y se mantiene en próximas visitas.
El idioma afecta: menú del sidebar, títulos de página, etiquetas de formulario, botones, encabezados de tabla y los títulos de los manuales de usuario.
El contenido de los manuales de usuario está escrito en español. Cambiar a EN traduce la estructura (títulos de sección, navegación) pero el texto descriptivo permanece en español.
Login y Sesión
IIniciar sesión
  1. Ir a la URL de la aplicación.
  2. Ingresar Username y Password.
  3. Clic en Sign In.
  4. Si las credenciales son correctas, se carga el dashboard.
CCerrar sesión
  1. Clic en el avatar/nombre en la esquina superior derecha del topbar.
  2. Seleccionar Logout.
  3. La sesión se cierra y se redirige al login.
EExpiración de sesión

El token de sesión expira automáticamente por inactividad. Cuando esto ocurra, el sistema mostrará un aviso y redirigirá al login. No se perderán datos — solo deberás volver a autenticarte.

Si ves el error "401 Unauthorized" al usar la aplicación, tu sesión expiró. Cierra el navegador e inicia sesión nuevamente.
Preguntas Frecuentes

¿Puedo cambiar mi propia contraseña?
Por ahora, el cambio de contraseña lo realiza un Admin desde la sección Users. Introduce la nueva contraseña en el campo correspondiente al editar el usuario.

¿Qué pasa si olvido la contraseña?
Contacta a un Admin o SuperAdmin para que restablezca tu contraseña desde la sección Users.

¿Por qué no veo el menú de Administration?
El menú de administración solo es visible para roles Admin y SuperAdmin. Si no lo ves, tu usuario tiene rol "Usuario".

¿Los cambios de tema afectan a todos los usuarios?
No. El tema de color se guarda en el navegador de cada usuario individualmente. Cada uno puede tener su propio tema.

¿Puedo tener múltiples SuperAdmin?
Sí. Puedes asignar el rol SuperAdmin a más de un usuario. Se recomienda tener al menos dos para no quedar sin acceso de administración.

¿Qué es "Stale Order Days" en Email Config?
Es el número de días que puede estar una orden de Aankoopbon en estado PENDING sin movimiento antes de considerarse "vencida". Sirve para alertas internas. El valor predeterminado es 4 días.

¿El logo se actualiza automáticamente para todos?
Sí. Al guardar un nuevo logo en Company Settings, todos los usuarios lo verán actualizado en el sidebar en su próxima carga o al limpiar la caché del navegador (Ctrl+Shift+R).

¿Por qué no veo el botón de eliminar usuario?
El botón solo aparece para usuarios inactivos. Primero desactiva el usuario con el botón , luego usa Show Inactive para verlo y podrás eliminarlo. Además, solo los usuarios con rol SuperAdmin tienen esta opción.

¿Qué diferencia hay entre permisos del rol y permisos individuales del usuario?
Los permisos del rol aplican a todos los usuarios que tienen ese rol. Los permisos individuales (botón en la fila del usuario) son overrides específicos para un usuario y tienen prioridad sobre el rol. Si un usuario no tiene overrides configurados, hereda completamente los permisos de su rol.

¿Puede un usuario sin rol Admin aprobar órdenes?
Sí. Marcando el permiso Approve en el rol del usuario (o en sus permisos individuales) para los sub-módulos Aankoopbonnen o Cost Calculation, ese usuario podrá aprobar y rechazar en esos módulos.
